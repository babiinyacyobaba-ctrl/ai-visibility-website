課題

AIに見つかる前に、
正しく伝わっているか

「東京でおすすめのホテルは？」「予算10万円の高級旅館は？」のような問いに対し、ユーザーは検索だけでなくAIの回答を参照するようになっています。

この変化の中で重要なのは、単に見つかることではなく、企業情報が正確かつ一貫して伝わることです。構造化データ、記述の整合性、情報の明確性が不足すると、AIやユーザーに誤って理解される可能性があります。

これは集客だけの話ではありません。社会に対する情報の信頼性の問題です。

AIの回答（情報整備あり）
帝国ホテル東京は1890年創業の老舗高級ホテルです。客室数は931室、東京都千代田区に位置し、チェックインは15:00からです。レストランは17店舗を有し、フレンチ・日本料理など多彩です。
AIの回答（情報整備不足）
〇〇旅館については詳細な情報が確認できませんでした。公式サイトをご確認ください。
仕組み

3ステップで
業界全体をスキャン

01

業界・競合を選択

ホテル・レストラン・ECなど業界を選択。知名度・高級・リスク順など抽出基準も指定できます。

02

AIスキャンを実行

各企業のWebサイトを自動取得・解析。meta情報・JSON-LD・スキーマ型・フィールド充実度を4軸で採点し、情報品質と可視性に関わる状態を整理します。

03

レポートを生成・活用

企業別PDFレポートと比較用データを生成。改善提案や現状把握の資料として活用できます。必要に応じてランキングHTMLも生成できます。

評価基準

4軸・100点満点の
スコアリング

0 — 30点

基本メタ

noindex・meta description・canonical・OGPタグなど、検索エンジンとAIへの基本情報を評価。情報の発見性と要約のしやすさに関わる基礎項目です。

0 — 20点

JSON-LD保有

企業名・住所・営業時間などをAIが読める形式（JSON-LD）で記述しているかを評価。情報の構造化は伝達品質に影響しうる重要な要素です。

0 — 30点

スキーマ型

JSON-LDの中でHotel・Restaurant・Productなど業界に適したスキーマ型が設定されているかを評価。種類の網羅性と業界適合性を採点します。

0 — 20点

フィールド充実度

スキーマの中に実際の値（住所・電話番号・料金・画像URLなど）がどれだけ埋まっているかを評価。定義だけあって中身が空では意味がありません。

よくある質問

FAQ

スコアはどのように計算されますか？
各企業の公式Webサイトを自動取得し、meta情報・JSON-LD構造化データ・スキーマ型・フィールド充実度の4軸を解析して100点満点で採点します。企業情報の整備状況と可視性に関わる要素を客観的に数値化した指標です。
診断に使用するデータはリアルタイムですか？
はい。スキャン実行時点での各企業Webサイトのデータをリアルタイムで取得・解析します。サイトが更新されれば次回のスキャンに反映されます。
どの業界に対応していますか？
現在、ホテル・旅館、レストラン・飲食、ECサイト、大手企業（コーポレートサイト）、汎用（メディア・不動産・求人等）の5業界に対応しています。
購入後はすぐにダウンロードできますか？
はい。決済完了後、ランキングHTMLおよびPDFレポートを即時ダウンロードいただけます。スキャン〜レポート生成まで通常数分以内に完了します。
診断対象の企業に通知されますか？
通知されません。診断は各企業の公開Webサイトへのアクセスのみで行うため、診断対象の企業には一切通知されません。
SemrushのAI Visibility Indexとは違うのですか？
測定対象が異なります。Semrushの同名指標はAI上での言及や露出の観測に近く、当サービスはWebサイト上の情報整備状況と可視性に関わる要素を診断します。両者は補完関係にあります。
AIでの掲載や集客成果は保証されますか？
保証しません。本サービスは公開Web上の情報整備状況と改善余地を診断するものであり、特定のAIサービスでの掲載、引用、順位、問い合わせ増加、売上増加等を保証するものではありません。
